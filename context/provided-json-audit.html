정규화 결정 필요

  • 파일명은 임시여도 학습 전 master manifest에는 canonical dataset version을 둔다.
  • landmark_id는 slug 형식으로 통일한다. 청와대는 cheongwadae로 진행한다.
  • id 정규화 시 file_name은 그대로 두고 original_image_id를 추가해 기존 매핑을 보존한다.
  • 서버 업로드 구조는 Datasets/<landmark_id>/labels.json과 Datasets/<landmark_id>/images/로 고정한다.
  • 각 landmark별 이미지 폴더는 충돌 방지를 위해 images/<landmark_id>/... 또는 unique filename으로 재배치한다.
  • confirmed만 기본 train에 넣고, uncertain/reject/not_landmark/duplicate는 별도 평가 또는 제외 정책으로 처리한다.
  • 중복 image_id/file_name은 학습 전 제거하거나 source group으로 묶는다.
