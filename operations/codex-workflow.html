Codex Workflow

작성일: 2026-05-13 · 대상: Sprint 1 모델 설계 및 학습 착수

운용 목적

Sprint 1에서는 모델 설계 결정을 빠르게 고정하고, 데이터 스냅샷, baseline, 첫 학습 실행, ONNX 배포 검증이 같은 문맥에서 이어지도록 Codex 역할을 분리한다. 핵심은 큰 결정을 깊게 검토하되, 파일 수정은 작고 검증 가능한 단위로 유지하는 것이다.

역할 분리

역할 사용 수준 책임
architect subagent GPT-5.5 xhigh 모델 구조, teacher/student 전략, 평가 기준, 위험한 tradeoff 검토.
worker GPT-5.5 medium 코딩과 문서 파일 수정. 소유 파일만 작게 변경하고 기존 스타일을 따른다.
researcher 필요 시 별도 역할 DINOv3, SigLIP2, EfficientNet-Lite, ONNX Runtime 등 최신 근거 확인.
verifier 필요 시 별도 역할 baseline, 학습 로그, ONNX export, 문서 링크 검증 결과를 독립적으로 확인.

진행 순서

  1. docs/goals/sprint1-model-training-goal.html에서 오늘의 성공 기준과 보류 질문을 먼저 고정한다.
  2. architect subagent가 설계안을 검토하고, 결정은 docs/decisions/ADR-0001-model-architecture.html에 남긴다.
  3. worker가 dataset contract, runbook, 학습 코드 같은 산출물을 소유 범위 안에서 수정한다.
  4. researcher는 시간이 지나면 바뀔 수 있는 모델, 라이브러리, 배포 제약을 현재 source로 확인한다.
  5. verifier는 실행 로그와 산출 파일을 기준으로 성공 주장과 미검증 항목을 분리한다.

문서 산출물

문서 용도
Goal Sprint 1 목표, 완료 조건, 의사결정 전제, 남은 리스크를 한 곳에 둔다.
ADR retrieval-first, closed-set head, teacher/student 선택처럼 되돌리기 어려운 결정을 기록한다.
Runbook 서버 확인, 데이터 동기화, baseline, fine-tuning, ONNX smoke test 절차를 실행 순서로 유지한다.
Pre-commit docs hook .githooks/pre-commit과 scripts/verify_docs.py로 문서 링크와 필수 문서 누락을 commit 전에 점검한다.

완료 기준

설계는 ADR에 기록되고, 학습 착수 절차는 runbook에 남아 있으며, 변경된 문서는 pre-commit docs hook 또는 그에 준하는 최소 검증으로 링크 깨짐이 없음을 확인한다. 검증하지 못한 항목은 완료 주장에 포함하지 않는다.
