Deploy

Vercel Docs Deploy

현재 문서는 빌드가 필요 없는 정적 HTML/CSS라 Vercel로 배포할 수 있다. Vercel의 Root Directory는 site로 설정한다. repo root에는 모델 학습용 pyproject.toml이 있으므로 root로 배포하면 Python 앱으로 잘못 감지될 수 있다.

Recommended

GitHub repo 연결

문서 폴더를 포함한 repo를 Vercel에 연결한다. Root Directory는 site, Framework Preset은 Other, Build Command는 비워둔다.

Local

Vercel CLI

로컬에서 바로 확인하려면 프로젝트 루트에서 vercel, 배포 확정은 vercel --prod를 실행한다.

배포 전 확인

  1. python scripts/verify_docs.py로 필수 HTML 문서와 링크를 확인한다.
  2. 브라우저에서 site/index.html과 site/project-brief.html를 열어 시스템 그림이 읽히는지 본다.
  3. Vercel 프로젝트의 Root Directory가 site인지 확인한다.
  4. 배포 URL에서 /, /project-brief.html, /operations/sprint1-status-2026-05-14.html를 확인한다.

주의

현재 배포 repo는 모델 학습 코드와 HTML 문서 사이트를 함께 포함한다. 모델 학습용 Python 설정이 repo root에 있으므로, Vercel Root Directory를 반드시 site로 둔다.
