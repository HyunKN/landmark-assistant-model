Model Serving Contract

작성일: 2026-05-13 · 대상: 모델 팀과 앱 팀 사이의 Sprint 1 인터페이스

역할 경계

모델 팀은 ONNX 모델, preprocessing spec, JSON prototype index, inference wrapper, Top-3 output schema를 제공한다. 앱 팀은 이미지 선택 UI, 상세정보 DB, 결과 화면, 안내 문구 표시를 담당한다. 양쪽의 계약은 landmark_id와 confidence fields를 기준으로 맞춘다.

Model Artifacts

파일 소유 설명
landmark_encoder.onnx 모델 팀 입력 이미지를 embedding 또는 logits로 변환하는 ONNX 모델.
prototype_index.json 모델 팀 landmark_id별 prototype embedding, class metadata, threshold를 포함. Sprint 1은 SQLite보다 단순한 JSON을 표준으로 둔다.
preprocessing.json 모델 팀 image size, RGB 변환, normalize mean/std, tensor layout.
labels_master.json 모델 팀 canonical landmark_id, name_ko, name_en, split, label policy를 포함하는 학습/앱 공통 manifest.
landmark_metadata.json 앱/콘텐츠 팀 상세정보 표시용 DB. 모델 성능에는 직접 영향 없음.

Input Contract

{
  "image_path": "local/path/to/user_image.jpg",
  "top_k": 3
}

범위 외 입력이면 status는 out_of_scope로 반환한다. 이때 앱 화면에는 Top-3를 표시하지 않고 안내 문구만 보여준다. 내부 분석을 위해 후보와 raw score는 debug log에만 저장한다.

Status Values

status 의미 앱 동작
matched 지원 범위 내 랜드마크로 판단. Top-3와 상세정보 이동 제공.
out_of_scope 지원 범위 밖 또는 confidence 부족. 지원 범위 외 안내 문구 표시. Top-3 후보는 화면에 숨기고 debug log에만 저장.
invalid_image 파일 decode 실패, 너무 낮은 해상도, 입력 형식 오류. 다른 사진 선택 또는 재촬영 안내.

Handoff Package

model_package/
  landmark_encoder.onnx
  preprocessing.json
  prototype_index.json
  labels_master.json

Python inference wrapper가 이 package를 기준으로 먼저 동작해야 한다. Flutter 앱은 같은 파일을 asset으로 받아 ONNX Runtime Mobile에 연결한다.
