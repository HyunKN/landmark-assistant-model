Landmark Assistant Project Brief

사진을 입력하면 온디바이스 모델이 지정 랜드마크 Top-3를 제시하고, 낮은 신뢰도 입력은 지원 범위 밖으로 안내하는 랜드마크 인식 어시스턴트다. Sprint 1은 실제 앱 형태보다 모델 탑재, 사진 입력, 검색, 결과 출력의 기술 시연을 우선한다.

13 현재 학습 대상 landmark class
4,457 confirmed training candidates
669 locked test records
S4 MobileCLIP2 Sprint 1 recognizer

왜 이 프로젝트를 하는가

단순 관광 정보 앱이 아니라, 제한된 랜드마크 집합 안에서 오프라인 동작, Top-k 후보, 신뢰도 기반 거절을 함께 보여주는 온디바이스 AI 시연이 목표다.

Primary

사진 기반 랜드마크 인식

사용자 사진을 MobileCLIP2-S4 image recognizer에 넣고 Top-3 후보와 confidence를 반환한다.

Trust

저신뢰 및 비대상 입력 거절

confidence와 margin을 로그로 남기고, 기준 미달 이미지는 지원 범위 밖으로 안내한다.

Next

자연어 검색 확장

"성의 외곽같은 돌담이 있는 공원" 같은 설명형 질의를 text encoder와 keyword engine으로 처리한다.

현재 확정된 방향

모델 선택, 학습 설정, 텍스트 검색 확장, 평가 방식을 분리해 기록한다.

항목결정근거 문서
Image recognizer MobileCLIP2-S4를 Sprint 1 단일 후보로 고정. MobileNetV4 Conv-AA Large IN12K는 fallback. ADR-0004
Training config batch 14, cosine scheduler + 2 epoch warmup, augmentation 완화, sqrt-weighted CE, per-class metric. ADR-0005
Text search 이미지 모델을 대체하지 않고 별도 text encoder / keyword engine으로 추가한다. ADR-0003
Evaluation Vision, reject, text retrieval, fusion, deployment를 분리해 평가한다. Selection Plan

Demo

시스템 그림

처음 만든 가독성 중심 다이어그램으로 복구했다. Sprint 1 범위와 Sprint 2 확장을 분리해 보여준다.

Input

사용자 입력

Sprint 1은 사진 입력이 핵심이다. 텍스트 입력은 다음 확장 범위로 분리한다.

  • 사진 업로드 또는 카메라 촬영
  • 자연어 질의: "돌담이 있는 공원"
On-device model

인식 및 검색 엔진

MobileCLIP2-S4를 중심으로 image embedding과 text embedding 경로를 운영한다.

  • MobileCLIP2-S4 image recognizer
  • Top-3 landmark 후보 산출
  • confidence / margin 계산
  • text encoder + keyword/BM25
Result

결과와 로그

화면은 사용자 친화적으로, 로그는 모델 개선에 필요한 수치 중심으로 남긴다.

  • Top-3 + 신뢰도 %
  • 지원 범위 밖 안내
  • 상세정보 연결
  • debug log 저장

Sprint 1 종료 기준: 사진 입력 -> on-device inference -> Top-3 또는 out-of-scope 안내까지 동작하는 데모.

팀원이 먼저 읽을 순서

처음 온 사람이 길을 잃지 않도록 읽는 순서를 고정한다.

  1. Sprint 1 Status Report - 지금 어디까지 왔는지.
  2. ADR-0004 - 왜 MobileCLIP2-S4인지.
  3. ADR-0005 - 서버 학습 설정이 왜 바뀌었는지.
  4. ADR-0003 - 자연어 검색을 어떻게 붙일지.
  5. Model Serving Contract - 앱 팀과 주고받을 입출력 계약.

주의할 점

현재 높은 성능 수치는 내부 split 기준이다. 실제 사용자 사진 일반화는 blind test, near-duplicate audit, out-of-scope set으로 별도 검증해야 한다. 발표에서는 "내부 test 기준"과 "실사용 검증 예정"을 구분한다.
